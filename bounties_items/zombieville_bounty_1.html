As said before, ZombieVille [1997] is a DOS game that was released by Psygnosis in an uncompleted state. It is considered abandon-ware as the game received negative initial reception and forgotten through the years.

Because of the nature of MS-DOS games are developed with 'in-house' methods and formats, file ripping isn't done by conventional means. My luck for trying to rip/view character sprites is basically measured within the following slide show:

I have used TiledGGD and TextureFinder 1.3.2 to help me find the character sprites. However, as seen in previous slide-show, I have more luck using said T.F. because it can generate the proper color pallet and proper resolution used in the game. The only file formats that I have been able to work with are .RAW* and .SPR. The .RAW* format tends to be a wild-card where it can be an image or a sound.

However, I have not been able to rip character sprites as seen from the first slide show above as of yet. I was only able to rip some static sprites that were used for backgrounds and the health "heads-up-display" screen.
I believe some of the character sprites might be stored in MATT_NXXX.ANM, or in .DAT files files. I'm not entirely sure since there's no easy way of seeing them nowadays.

It is generally said that if a game cannot have its sprites ripped from the game, then the next step is to screen-shot and paste all sprites into a single sheet. The reason why I would not proceed with this method is because of the following:

  • The character sprites are made to shrink in size the farther away the character is on screen.
  • Not all characters are programmed to be allowed to be up-close.
  • Not all characters' actions are programmed to be viewed in all 16 directions.
  • Characters in this game (both hostile and not) have 16 directions of animations.
  • There could be unused characters and/or move-sets stored somewhere in the files.
