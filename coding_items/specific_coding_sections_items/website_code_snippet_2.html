Chrome Iframe Rendering Fix
Website Code Snippet(s)

This website was primarily designed to use using NotePad++ and Google Chrome. With this simple combination, I've managed to overcome many issues through trial and error processes.

I've used the Iframe component to display majority of contents that you may have seen throughout many pages. Using Iframes have their ups and downs

I recall one unusual occurrence when I was testing my Iframe content within web pages with computer Chrome browser.

This infrequent occurrence resulted that the Iframe content was somehow rendered invisible yet functional at the same. The content was able to be interacted with but it did not show up on screen unless the screen was RESIZED or REFRESHED.

This was the most unusual for me because I wasn't sure what was causing this issue with Chrome.
So I've attempted some work-arounds and I have found the following code snippet that fixed my Chrome Iframe rendering problem.

test_hidden_fix()
{
let userAgentString = navigator.userAgent;
let chromeAgent = userAgentString.indexOf("Chrome") > -1;
if (chromeAgent == true)
  {
   window.location.visibility = "hidden";
   window.location.visibility = "visible";
  }
}

There are two parts at work within the previous code snippet.
This function is called when Iframe content has finished loaded. The first part is the IF STATEMENTS. The variables basically assign the value of what kind of browser is being used to access the Iframe contents.

If the browser that's currently being used is actually Google Chrome, then proceed with the 2nd component.

The second component basically renders the Iframe content "invisible", despite the content existing already.
Then, the Iframe content is rendered visible once again.

End Result

The end result yielded in the following solution: Google Chrome will no longer randomly render my Iframe content invisible without the need of resizing the screen.

Despite this workaround providing me some website relief, it comes with one downside. Longer "load" times.

The function doesn't check to see if the Iframe content is accidentally rendered invisible because said content is considered to be visible.
Not only that, but if you click around and drag an element from an 'invisible' page, one can see that elements do exist and function.

The only simple way to make the Iframe content rendered 100% of the time without resizing the screen was to use the previously shown function.
