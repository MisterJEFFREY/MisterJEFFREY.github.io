Hot Potato
Game Project

Game Play Screen Shots

This particular 'resource-light' game project was intended to be used in a workshop at a a University under the Computer Science Club.
The game was designed to showcase certain components of game-design, such as object inheritance, proper asset usages, and AI attempts.

The game-play consists of one player that controls the character on the left side of the screen.
The objective is to attempt to hammer the bomb outside of the screen until the timer runs out.

Controls for this game as follows:

    Menu Keys
  • P Key: Start the Game
  • Q Key: Quits the game
    • Game Play Keys
  • A Key: Move Left
  • D Key: Move Right
  • J Key: Swing Hammer
  • R Key: Restart Match

This particular game was built within about 2 & 1/2 days.
Reason for this was because a workshop was requested at a last minute notice. This meant that I had to juggle between a full time academic course load, create and fully design this project, and advertise within a WEEK.
Despite the quick work-arounds, the project was able to be produced and showcased in the workshop to interested peers.
The downside of this quick project was that there was not enough time to fully flesh out the opponent's A.I. to take into account for all game-play scenarios. (I.e, What if the bomb is moving too fast on the opponent's side?)
